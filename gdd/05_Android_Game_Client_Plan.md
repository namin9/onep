# [GDD] 05. 안드로이드 게임 클라이언트 개발 계획

이 문서는 '아카이브 에디터'의 안드로이드 게임 클라이언트 개발을 위한 구체적인 계획과 에셋 전략을 포함합니다.

---

## 1. 에셋 계획 (사용자 피드백 반영):

### I. 시각적 스타일:

*   **기본 캐릭터/몬스터:** 단순하고 깔끔한 디자인 (예: 치비/스타일라이즈드 2D). 최소한의 프레임으로 애니메이션하기 쉽게 제작합니다.
*   **캐릭터 스킨:** **고품질의 매력적인 2D 일러스트.** 게임의 시각적 매력과 진행의 핵심 요소가 됩니다. 여기에 가장 많은 예술적 노력을 집중합니다.
*   **UI:** 깔끔하고 기능적이며, 스킨 일러스트가 돋보이도록 모던한 다크 테마를 유지합니다.

### II. 핵심 에셋 카테고리 및 예상 필요량:

1.  **UI 요소 (PNG/SVG, 일관된 해상도):**
    *   **버튼:** 일반 버튼 + 기능별 버튼 (뽑기, 업그레이드, 환생 등) - 3-5가지 스타일, 5-10가지 아이콘.
    *   **아이콘:** 재화, 스탯, 메뉴/내비게이션, 일반 아이템/스킬 플레이스홀더 - 총 30-50개.
    *   **배경:** 메인 게임/전투 배경 (단순), 메뉴/모달 배경 - 총 2-5개.
    *   **진행 바:** 2-3가지 스타일.
    *   **질감/패턴/프레임:** 단순하고 시선을 분산시키지 않는 형태 - 총 10-20개.

2.  **캐릭터/몬스터 스프라이트 (PNG, 화면 최적화):**
    *   **플레이어 기본 캐릭터:**
        *   **아이들(대기):** 1-3 프레임.
        *   **공격:** 2-4 프레임 (간단하고 정면 공격).
        *   **피격/스킬:** 선택 사항, 1-2 프레임.
    *   **몬스터 (도서관 용):**
        *   **아이들(대기):** 몬스터당 1-2 프레임 (정적이거나 매우 미세한 루프).
        *   **총계:** 시작 시 5-10개 고유 몬스터 스프라이트 (단순 디자인).

3.  **뽑기 스킨 (PNG, 고품질 일러스트):**
    *   **스킨 아트:** **20-50개 고유 스킨 이미지 (높은 퀄리티, 다양한 매력).** 여기에 주요 아트 작업과 예산을 집중합니다. 각 스킨은 독립적이고 매력적인 일러스트 형태로 제작됩니다.
    *   **희귀도 시각화:** SSS/S/A 등급 스킨에 대한 미묘한 시각적 효과.

4.  **이펙트 (PNG 스프라이트 시트, 간단한):**
    *   **공격 이펙트:** 1가지 간단한 이펙트.
    *   **업그레이드 이펙트:** 1가지 간단한 이펙트.
    *   **뽑기 이펙트:** 1가지 간단하지만 "일러스트 등장"을 강조하는 효과.

5.  **폰트 (TTF/OTF):** 1-2가지 커스텀 폰트.

### III. 에셋 제작 워크플로우:

1.  **디자인 철학:** 핵심 캐릭터는 단순하게, 스킨은 풍부한 일러스트 중심으로 디자인합니다.
2.  **우선순위:** 고품질 스킨 일러스트와 필수 UI 요소부터 우선 제작합니다.
3.  **반복:** 플레이스홀더를 사용하다가, 완성된 스킨 일러스트가 준비되는 대로 통합합니다.
4.  **확장성:** 스킨 일러스트는 높은 기본 해상도(예: `xxxhdpi` 기준)로 디자인하여 다양한 기기에서 선명하게 보이도록 합니다.

## 2. 네이티브 안드로이드 앱 개발 TODO 리스트 (Kotlin/Java):

1.  **안드로이드 프로젝트 초기 설정:**
    *   안드로이드 스튜디오에서 새로운 안드로이드 프로젝트를 생성합니다 (Kotlin 또는 Java 선택).
    *   네트워크 통신(Retrofit), JSON 파싱(GSON/Moshi), 이미지 로딩(Coil/Glide), 의존성 주입(Koin/Hilt 등, 필요시)을 위한 라이브러리 의존성을 설정합니다.
    *   **에셋 파이프라인 통합:** 다양한 화면 밀도를 처리하고 초기 플레이스홀더 에셋을 포함할 수 있도록 `res/drawable`, `res/layout`, `res/values`를 준비합니다.

2.  **인증 모듈 개발:**
    *   로그인 화면 UI를 디자인합니다 (모의 Google 코드 입력, 시작 버튼, 상태 표시 등).
    *   `/api/user/login` API 호출 로직을 구현합니다.
    *   JWT를 안전하게 저장합니다 (예: EncryptedSharedPreferences).

3.  **메인 게임 화면 UI 설계:**
    *   코어 `Activity`/`Fragment` 레이아웃을 디자인합니다 (플레이어 정보 사이드바, 메인 게임 콘텐츠 영역, 업그레이드, 뽑기, 도서관, 영혼과 같은 다양한 섹션을 위한 내비게이션 탭/버튼).
    *   반응형 디자인을 위해 `ConstraintLayout` 또는 `LinearLayout`을 사용합니다.
    *   **플레이스홀더 에셋 통합:** UI 구조를 구축하기 위해 버튼, 아이콘, 배경 등에 초기 플레이스홀더 에셋을 사용합니다.

4.  **데이터 관리:**
    *   백엔드 데이터 형식(UserStats, Skin, Monster, Talent 등)에 맞춰 데이터 모델(Kotlin 데이터 클래스/Java POJO)을 정의합니다.
    *   UI를 위한 로컬 상태 관리 방식(예: Jetpack Compose/XML 뷰를 위한 ViewModel/LiveData)을 구현합니다.

5.  **API 통합 계층 구현:**
    *   모든 백엔드 API 엔드포인트를 위한 Retrofit 서비스 인터페이스를 생성합니다.
    *   인증된 요청에 JWT를 자동으로 추가하는 `AuthInterceptor`를 구현합니다.

6.  **게임 루프 및 방치형 진행:**
    *   주기적인 `/api/game/progress` 동기화를 위한 백그라운드 작업(예: `WorkManager` 또는 `Service`)을 구현합니다.
    *   리소스 카운터를 업데이트하기 위한 클라이언트 측 시각적 루프(`Handler`/`Runnable` 또는 코루틴)를 구현합니다.
    *   현재 스테이지, 리소스, 초당 예상 획득량을 표시하는 UI를 구현합니다.

7.  **업그레이드 섹션:**
    *   스탯 레벨 표시 및 업그레이드 버튼 UI를 구현합니다.
    *   `/api/user/upgrade_stat` 호출 및 UI 업데이트 로직을 구현합니다.
    *   **비용 표시:** 업그레이드에 필요한 루비 비용을 표시합니다.

8.  **뽑기 섹션:**
    *   1회/10회 뽑기 버튼 UI를 구현합니다.
    *   `/api/gacha/pull` 호출 로직을 구현합니다.
    *   획득한 **고품질 스킨 일러스트**와 그 세부 정보(등급, 옵션)를 표시합니다.
    *   **천장 카운터(Pity Counter) 표시:** 선택적으로 천장 카운터를 표시합니다.

9.  **도서관 섹션:**
    *   몬스터 도감(`RecyclerView` 또는 `ListView`)을 표시하는 UI를 구현합니다.
    *   `/api/library/codex` 호출 로직을 구현합니다.
    *   **단순한 몬스터 스프라이트** 및 수집된 조각 진행 상황을 표시합니다.
    *   `handleSubmitFragments` 호출을 위한 UI를 구현합니다 (예: 조각 제출 테스트용).

10. **영혼 시스템 섹션:**
    *   환생 버튼 UI를 구현합니다.
    *   특성 트리(`RecyclerView`/커스텀 뷰)를 표시하는 UI를 구현합니다.
    *   `/api/soul/rebirth`, `/api/soul/unlock_talent`, `/api/soul/talents` 호출 로직을 구현합니다.
    *   **특성 세부 정보**를 표시하고 해금된 특성을 강조합니다.

11. **사용자 피드백 및 에러 처리:**
    *   사용자 피드백 및 에러 표시를 위한 `Toast` 메시지, `Snackbar`, `AlertDialog` 등을 구현합니다.
    *   API 호출 시 로딩 표시기를 구현합니다.

12. **에셋 통합 (지속):**
    *   고품질 스킨 일러스트 및 기타 에셋이 준비되는 대로 앱의 `res/drawable`에 통합하고 UI에 사용합니다.
